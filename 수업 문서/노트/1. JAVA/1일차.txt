컴퓨터 메모리 구조

	1. Stack : 자바의 구조★. 쌓다는 의미
		Stack(쌓다) vs Queue(줄, 열)
			1) Stack은 1,2,3 순서로 쌓으면 3,2,1 순으로 나온다.
			(First In Last Out / Last In Fist Out)
			2) Queue는 1,2,3 순서로 쌓으면 1,2,3 순으로 나온다.
			(Fist In First Out / Last In Last Out)
			3) 자바는 스택구조로 main이 가장 먼저 쌓이고 마지막으로 종료된다.
	2. Heap
	3. Static



변수 : 어떠한 데이터를 집어넣을 공간. 재사용과 관리의 용이성을 위해 사용. 하나의 값만 사용 가능

int a; => 메모리 중 stack 공간에 int자료형의 a라는 영역을 선언하는것

자료형 : 어떤 종류의 데이터가 들어갈지 지정해 주는것
	1. 기본자료형 : boolean, char, byte, short, int(정수 기본형. 컴퓨터는 4바이트 계산이 가장 용이하여 기본형으로 채택),
		 long, float, double(실수 기본형, 정확도를 위해[소숫점 15자리까지 정확도 확보] 8바이트지만 기본형으로 채택)
	2. 참조형자료형 : 기본자료형 8항목 이외에는 모두 참조형 자료형이다. 그러니까 기본자료형 8개만 외우자

1byte = 8bit = 8자리의 2진수 바이트숫자 들어감

'=' -> ★오른쪽의 값을 왼쪽에 집어 넣겠다
ex) age = count;  -> count 안에 있는 값을 왼쪽의 age에 넣겠다. (들어갈 수 있는 자료형 한정)

가끔 기본값이 아닌 쓰레기값이 변수안에 들어갈 때가 있어서 사용 전 초기화는 필수다.
